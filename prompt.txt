あなたは私の開発アシスタントです。
役割は「Supabase のスキーマ変更に合わせて、TypeScript 型定義を最新化すること」です。

前提状況:
- Supabase データベース側では、旧テーブル `ideas` / `tasks` / `recurring_tasks` および不要関数をすでに削除済み。
- 現在残っているテーブルは以下のとおり:
  - done
  - recurring_templates
  - subtasks
  - unified_tasks
  - user_metadata
  - user_settings
- 使用中の関数:
  - handle_updated_at
  - generate_display_number
  - update_updated_at_column

やってほしいこと:
1. Supabase CLI を用いて、最新のスキーマから TypeScript 型 (`supabase-new.ts`) を再生成してください。
   - コマンド:  
     `npx supabase gen types typescript --project-id <PROJECT_ID> --schema public > src/types/supabase-new.ts`
2. 型ファイル内に、削除済みテーブルや不要関数が残っていないことを確認してください。
3. 新しい型定義を `src/types/supabase-new.ts` に反映して commit してください。
4. 結果として、型ファイルの差分（削除された型や関数の定義）が確認できるようにしてください。

出力フォーマット:
- ✅ 削除されている型や関数の一覧
- ✅ 新しく残っているテーブル・関数の型一覧（要約）
- ✅ commit ハッシュと差分の概要
